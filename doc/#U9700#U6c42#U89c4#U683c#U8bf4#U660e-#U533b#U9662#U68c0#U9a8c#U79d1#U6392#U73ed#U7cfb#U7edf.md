## 2. 业务背景

检验科排班具有以下特点：

- 员工按小组管理，不能跨组随意排班
- 夜班通常由外部表格先排定
- 夜班后需要自动带出连续休息约束
- 日班岗位存在默认人数、按星期覆盖、按日期覆盖等多层规则
- 同时需要兼顾公平性与业务刚性约束

因此系统需要支持“配置规则 + 导入夜班 + 自动生成日班 + 导出结果”的完整闭环。

## 3. 用户角色与使用场景

### 3.1 用户角色

- 排班员：维护基础数据、导入夜班、生成并导出排班
- 科室管理人员：查看排班结果、核对规则配置

### 3.2 典型流程

1. 维护小组、角色、员工、岗位。
2. 维护月度休息目标和个人休息计划。
3. 导入当月夜班 Excel。
4. 配置特殊规则、星期需求和日期需求。
5. 执行排班预检查。
6. 生成排班并查看备注。
7. 导出 Excel 用于存档或打印。

## 6. 业务约束

### 6.1 排班边界

- 系统当前只自动生成日班，不自动生成夜班
- 日班员工池限定在小组内启用员工
- 夜班数据以导入结果为准
- 排班结果按月覆盖写入

### 6.2 休息约束

- 当前月夜班后休息默认影响 `D+1`、`D+2`
- 上月末夜班可能影响本月前几天
- 固定休息日优先级高于日班安排
- 员工休息目标由角色目标和个人计划共同决定

### 6.3 公平性目标

- 尽量平衡员工工作量
- 尽量平衡周末休息
- 尽量让实际休息天数接近目标

## 7. 接口与数据输出要求

### 7.1 基础接口分类

- 基础数据接口：小组、角色、员工、岗位、专业
- 规则接口：特殊规则、月度约束、休息计划、岗位需求覆盖
- 夜班接口：夜班导入、夜班查询
- 排班接口：预检查、生成、查询、备注查询、导出

### 7.2 输出要求

- 排班结果按“日期 + 员工 + 班次”输出
- 导出文件需支持 Excel 打开
- 备注说明需可追溯跨月与欠休情况

## 8. 验收要点

1. 能成功维护小组、角色、员工、岗位等基础数据。
2. 能按月份导入夜班 Excel 并覆盖原有数据。
3. 若夜班员工不存在或不可上夜班，系统应阻止导入或生成。
4. 排班预检查能提示某些日期需求人数超过可用人数。
5. 生成结果中，夜班后休息日不会被再次排岗。
6. 固定休息日不会被排岗。
7. 能正确处理跨月夜班带来的本月休息。
8. 能导出 Excel，且导出内容与系统结果一致。
9. 有备注时，导出文件包含备注工作表。

## 9. 当前差异说明

以下内容在旧版本需求中出现过，但当前代码并未完整实现，因此不建议继续写成已上线能力：

- 节假日单人夜班规则
- 复杂权限体系
- 系统内换班审批流程
- 更高级别的全局最优求解器
