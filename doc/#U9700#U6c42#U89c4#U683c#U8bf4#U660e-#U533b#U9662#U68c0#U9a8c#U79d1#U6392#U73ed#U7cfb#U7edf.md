# 医院检验科排班系统 — 需求规格说明

## 1. 引言

### 1.1 目的

本文档汇总医院检验科多组排班与夜班导入类系统的**功能与非功能需求**，供开发、测试与验收使用。

### 1.2 名词解释

| 名词 | 定义 |
|------|------|
| 夜班开始日 | 夜班于当日 17:30 开始的公历日期，系统在表内记为该日日期的夜班 |
| 春节区间 | 按公历配置的日期范围；该范围内夜班人数为 1，范围外为 2 |
| 出班 | 上完指定夜班后的首个全日无岗日（业务上与禁排联动，见 3.3） |
| 组 | 临检、生化免疫、微生物等专业组，数据隔离 |

### 1.3 参考文档

- `方案-医院检验科排班系统.md`（建设方案）

## 2. 用户与场景

### 2.1 用户角色（一期可合并）

- **排班员/秘书**：维护基础数据、导入夜班、排日班、导出。
- **科室管理**（可选）：审核或查看（二期权限细化）。

### 2.2 典型场景

1. 年初或春节前配置当年（或多段）**公历春节区间**。
2. 各组维护本组人员与班种、周规则。
3. 领导在 Excel 中排好夜班，排班员**导入**系统。
4. 系统校验夜班人数与人员资格，并生成**禁排日期**。
5. 各组在月历上排**日班**，系统阻止或提示违反禁排、必须休、人数不足等情况。
6. **导出**月度表存档或打印。

## 3. 功能需求

### 3.1 组织与组

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-ORG-01 | 支持维护多个专业组（名称、排序、是否启用） | P0 |
| FR-ORG-02 | 人员必须属于且仅属于一个组 | P0 |

### 3.2 人员管理

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-STAFF-01 | 维护人员：姓名、工号（可选）、所属组、出生日期或年龄（可选，用于展示/扩展规则） | P0 |
| FR-STAFF-02 | 支持标记 **免夜班**（该人员不应出现在夜班导入中） | P0 |
| FR-STAFF-03 | 支持标记 **参与夜班**（或与免夜班互斥，具体实现二选一并在界面说明） | P0 |
| FR-STAFF-04 | 仅本组可见/可编辑本组人员（一期可与全局维护合并，但数据上归属组唯一） | P1 |

### 3.3 班种与周规则

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-SHIFT-01 | 每组可配置多个班种（如 A1、A2、A3），含名称、排序 | P0 |
| FR-SHIFT-02 | 每个班种可配置「每班需要人数」或等价字段 | P0 |
| FR-SHIFT-03 | 支持配置「周一～周日」默认：某日需要哪些班种及各几人 | P0 |
| FR-SHIFT-04 | 支持覆盖规则：指定「周几」使用特殊班种组合/人数（与默认合并或替换，需在界面约定一种模式） | P1 |

### 3.4 春节与全局配置

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-HOL-01 | 支持按 **公历年** 维护一段或多段「春节夜班按 1 人」的日期区间 | P0 |
| FR-HOL-02 | 非区间内夜班导入时每行 **2 人**；区间内每行 **1 人** | P0 |

### 3.5 夜班（仅导入，不自动排班）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-NIGHT-01 | 系统**不提供**夜班自动排班功能 | P0 |
| FR-NIGHT-02 | 提供标准 **Excel 导入模板**（至少含：夜班开始日、人员1、人员2；春节区间为人员1 或人员2 为空，规则与 FR-HOL-02 一致） | P0 |
| FR-NIGHT-03 | 导入时 **日期列 = 夜班开始日**（与方案一致） | P0 |
| FR-NIGHT-04 | 导入校验：人数与春节规则一致；人员重复、空行策略可配置或按模板约定 | P0 |
| FR-NIGHT-05 | 导入校验：夜班人员不得为「免夜班」人员；若配置「仅参与名单可上夜班」，则须在名单中 | P0 |
| FR-NIGHT-06 | 支持查看当月夜班列表；支持在权限下删除/重新导入覆盖（策略需明确：全量覆盖或按日覆盖） | P1 |

### 3.6 出班链禁排

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-REST-01 | 对夜班开始日 **D** 的当班人员，**D+1、D+2** 日不允许排任何班（日班、夜班） | P0 |
| FR-REST-02 | **D+3** 默认不允许排班，但允许排班员对该日执行「允许排班」操作（需留痕：操作人、时间） | P1 |
| FR-REST-03 | 多夜班重叠时，禁排区间合并，取最严规则 | P0 |

### 3.7 日班排班（按组）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-DAY-01 | 按月选择年月，按组展示月历或表格 | P0 |
| FR-DAY-02 | 日班只能从**本组人员**中选择，**不允许跨组** | P0 |
| FR-DAY-03 | 排班时若违反 FR-REST-xx 或「必须休息」，系统应**阻止保存**或**明确报错**（一期建议阻止） | P0 |
| FR-DAY-04 | 班种人数与周规则可作为「参考」展示或弱提示（一期可仅展示目标人数） | P2 |

### 3.8 个人例外

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-EX-01 | 支持为指定人员指定公历日期标记为 **必须休息**（该日不可排班） | P0 |
| FR-EX-02 | （可选）支持「希望休息」仅提示不硬拦，或合并为必须休，产品二选一 | P2 |

### 3.9 换班

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-SWAP-01 | **不提供**换班申请、审批流程；**不要求**在系统中记录私下换班 | P0 |

### 3.10 导出

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-EXP-01 | 支持导出当月 **Excel**，至少包含：夜班一览（开始日、人员）；各组日班表 | P0 |
| FR-EXP-02 | 导出列与导入模板兼容，便于再次编辑导入（夜班部分） | P1 |

## 4. 非功能需求

| ID | 类别 | 需求描述 | 优先级 |
|----|------|----------|--------|
| NFR-01 | 部署 | 提供 Windows **安装包**，安装后可在本机运行，默认 **127.0.0.1** 访问 | P0 |
| NFR-02 | 数据 | 使用本地数据库（建议 SQLite），数据文件路径可配置或可说明默认路径 | P0 |
| NFR-03 | 性能 | 单科室规模（人员 ≤500、月排班单元格量级）操作响应可接受（具体指标可在测试中定：如常规操作 &lt;2s） | P1 |
| NFR-04 | 安全 | 一期至少提供简单访问控制（如登录密码或单机免登录，需书面确认） | P1 |
| NFR-05 | 备份 | 提供或说明通过复制数据库文件的备份方式；二期可做备份向导 | P1 |

## 5. 约束与假设

- **夜班数据来源**：仅以导入为准，系统不生成夜班。
- **日期口径**：夜班记录与校验均以 **夜班开始日** 为 D。
- **春节**：仅按 **公历日期区间** 配置，不做农历自动换算。
- **跨组**：日班人员池不跨组。

---

## 6. 验收要点（摘要）

1. 配置春节区间后，导入夜班在非春节为 2 人、春节为 1 人，错误数据被拒绝并提示原因。
2. 导入夜班后，对应人员在 D+1、D+2 无法被排入任何班；D+3 默认不可排，授权后可排。
3. 各组日班只能选择本组人员；跨组不可选。
4. 免夜班人员无法作为夜班导入通过校验。
5. 必须休息日与禁排日同时满足时，排班仍不可保存。
6. 可导出 Excel，内容与界面月度数据一致。
